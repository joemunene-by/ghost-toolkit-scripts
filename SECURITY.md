# Security Policy

## 🔐 Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 4.0.x   | ✅ |
| 3.0.x   | ✅ |
| < 3.0   | ❌ End of life |

## 🚨 Reporting a Vulnerability

If you discover a security vulnerability in Ghost Toolkit, please report it responsibly:

1. **Do NOT** create a public GitHub issue
2. **Email** the maintainer directly: joemunene-by
3. **Include**:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (optional)

## ⚡ Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial Response**: Within 7 days
- **Fix Timeline**: Depending on severity

## 📜 Scope

This security policy applies to:
- Ghost Toolkit Scripts
- Tool wrapper scripts
- Infrastructure code (Docker, Terraform)

**Out of Scope:**
- Third-party tools (nmap, sqlmap, etc.)
- Target systems during pentesting

## 🔑 Responsible Disclosure

We follow responsible disclosure guidelines. Please give reasonable time for fixes before public disclosure.
